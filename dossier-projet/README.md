# Dossier de Projet — Reminder Famille

Document technique présentant le projet (**20 pages maximum**).

## Plan (basé sur la Checklist Préparation Soutenance DWWM)

1. **Introduction générale** — contexte, problème, hypothèse
2. **Présentation du projet** — besoin, marché, historique
3. **Équipe et gestion de projet** — méthodologie (solo, agile-light)
4. **Spécifications fonctionnelles** — use cases, user stories
5. **Spécifications techniques** — stack, environnement, outils
6. **Maquettage** — charte graphique, wireframes, mockups
7. **Intégration** — responsive, HTML/CSS, media queries
8. **Conception BDD** — MCD, MLD, MPD, cardinalités, ACID
9. **Mise en place BDD** — CREATE TABLE, FOREIGN KEY, contraintes
10. **Mission 1 — Authentification + Familles** (CRUD + permissions)
11. **Mission 2 — Tâches + Validation parent** (API + AJAX-style)
12. **Mission 3 — Export iCal** (intégration standard externe)
13. **Déploiement** — environnements, .env, Docker (futur)
14. **Sécurité** — SQL injection, XSS, CSRF, hash, contrôle d'accès
15. **Conclusion générale** — bilan, formation, suite

## Sous-dossiers

- `diagrammes/` — UML (packages, use cases), Merise (MCD/MLD/MPD), activité, séquence
- `maquettes/` — wireframes, mockups, charte graphique
